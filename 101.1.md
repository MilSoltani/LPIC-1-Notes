# 101.1 Hardwareeinstellungen ermitteln und konfigurieren

- PC-Konfigurationsprogramm: beim Einschalten der Maschine aufgerufen werden kann:
  - BIOS (Basic Input/Output System): einem Firmwarestandard, der typische Konfigurationsroutinen für x86-Hauptplatinen
  - UEFI (Unified ExtensibleFirmware Interface): mächtigere Funktionen für die Erkennung, Funktionsprüfung und Konfiguration von Hardware sowie die Aktualisierung von Firmware

- Zwei Wege, die von Linux erkannte Hardware anzuzeigen:
  - Befehle (LSPCI, LSUSB)
  - Dateisystemen (/proc, /sys, /dev)

## 1- Befehle

### lspci

- Zeigt alle Geräte an, die aktuell an den PCI-Bus (Peripheral Component Interconnect) angeschlossen sind:
- Bei PCI-Geräten handelt es sich entweder um:
  - Komponenten, die an die Hauptplatine angeschlossen sind (z.B. Festplatten-Controller)
  - Erweiterungskarten, die in einen PCI-Steckplatz eingesteckt ist (z.B. eine externe Grafikkarte)
- Ausgabe des Befehls:
  - Die hexadezimale Zahl am Anfang jeder Zeile ist die eindeutige Adresse des jeweiligen PCI-Geräts

```bash
lspci
# ...
# 04:02.0 Network controller: Ralink corp. RT2561/RT61 802.11g PCI
# ...

lspci -s 04:02.0 -v      # mehr Info über das spezifische Gerät
# 04:02.0 Network controller: Ralink corp. RT2561/RT61 802.11g PCI
#    Subsystem: Linksys WMP54G v4.1         -> Marke und Modell des Geräts
#    Flags: bus master, slow devsel, latency 32, IRQ 21
#    Memory at e3100000 (32-bit, non-prefetchable) [size=32K]
#    Capabilities: [40] Power Management version 2
#    kernel driver in use: rt61pci         -> Das zugeordnete Kernelmodul

lspci -s 01:00.0 -k      # welches Kernelmodul für das angegebene Gerät verwendet wird?
# 01:00.0 VGA compatible controller: NVIDIA Corporation GM107 [GeForce GTX 750 Ti] (rev a2)
#    kernel driver in use: nvidia
#    kernel modules: nouveau, nvidia_drm, nvidia
```

- Eventuell sind nicht allen Geräten entsprechende Module zugeordnet
  - bestimmte Geräte können von Anwendungen direkt angesprochen werden, ohne dass ein Kernelmodul dazwischengeschaltet ist

### Kernelmodul

- jedes Gerät benötigt eine Softwarekomponente (Kernelmodul), mit der es gesteuert wird
  - werden auch als Treiber (Driver) bezeichnet  
  - Teil des offiziellen LinuxKernels
  - oder separat aus anderen Quellen bezogen
- Bereits beim Laden des Kernels werden zahlreiche Module mitgeladen

- Der Befehl lsmod zeigt alle aktuell geladenen Module an
  - Module: Name des Moduls
  - Size: Arbeitsspeicher (in Bytes), der vom Modul belegt wird
  - Used by: Abhängige Module.
    - Einige Module benötigen andere Module, um korrekt zu funktionieren.
 
```bash
lsmod
# Module                  Size  Used by
# sd_mod                 81920  0
# uinput                 20480  1
# intel_rapl_common      53248  1 intel_rapl_msr
# kvm_amd               221184  0
# ccp                   163840  1 kvm_amd
# kvm                  1396736  1 kvm_amd
```

## 2- Informationsdateien und Gerätedateien
- Die Befehle lspci, lsusb und lsmod bieten einen benutzerfreundlichen Weg, die vom Betriebssystem gespeicherten Hardwareinformationen abzurufen
- Diese Informationen sind in speziellen Dateien in den Verzeichnissen **/proc** und **/sys**
  - Sie verweisen auf spezielle Dateisysteme, die nicht auf Speichergeräten liegen, sondern in einem RAM-Bereich,
  - Der Kernel liegt in diesen Verzeichnissen die Konfigurationseinstellungen sowie Informationen über laufende Prozesse ab
  - Solche Dateisysteme sind nicht für die normale Dateispeicherung gedacht, daher werden sie **Pseudo-Dateisysteme** genannt
  - /sys enthält vor allem Geräteinformationen und Hardwaredaten des Kernels
  - /proc Kerneldatenstrukturen abbildet, zu denen auch Daten über die laufenden Prozesse gehören
    - /proc/cpuinfo
      - Enthält genaue Informationen über die vom Betriebssystem erkannte(n) CPU(s)
    - /proc/interrupts
      - Enthält eine Tabelle mit der Anzahl der Unterbrechungen, die pro Eingabe/Ausgabe-Bausteinfür jede CPU aufgetreten sind
    - /proc/ioports
      - Enhält eine Liste der aktuell registrierten Port-Bereiche für die Ein- und Ausgabe
    - /proc/dma
      - Gibt Auskunft über die vom System verwendeten DMA-Kanäle (Direct Memory Access)
  - /dev:
    - Jede Datei in /dev stellt ein Gerät dar
    - Dazu gehören insbesondere Speichergeräte: /dev/hda
      - Die einzelnen Partitionen dieser Festplatte sind dabei als /dev/hda1, /dev/hda2 usw. dargestellt
    - Der Linux-Kernel erkennt neu angeschlossene Hardware und übergibt die entsprechenden Informationen an den udev-Prozess
    - udev-Prozess dann das Gerät identifiziert und dynamisch entsprechende Dateien unter /dev anlegt
    - udev konfiguriert:
      - die im laufenden Betrieb erkannten Geräte (hotplug detection)
      - auch die bereits beim Einschalten des Rechners vorhandenen Geräte (coldplug detection)
    - Wird ein neues Gerät erkannt, sucht udev im Verzeichnis /etc/udev/rules.d/ nach einer passenden Regel dafür
    - Die wichtigsten Regeln werden von den Distributionen zur Verfügung gestellt
    - Für besondere Fälle können neue Regeln hinzugefügt werden.
    - Blockgeräte
      - Speichergeräte werden in Linux als **Blockgeräte** behandelt und im Verzeichnis `/dev` als Dateien dargestellt
      - Ältere Geräte nutzen Namen wie `/dev/hd*` (IDE-Laufwerke) oder `/dev/fd*` (Diskettenlaufwerke)
      - Moderne Speichergeräte (IDE, SATA, SSD, USB) verwenden meist das Präfix `/dev/sd*`
      - Laufwerke werden mit Buchstaben gekennzeichnet: z. B. `/dev/sda`, `/dev/sdb`
      - Partitionen werden nummeriert: z. B. `/dev/sda1`, `/dev/sda2`, `/dev/sdb1`
      - SD-Karten nutzen das Schema `/dev/mmcblkXpY`, z. B. `/dev/mmcblk0p1`
      - NVMe-SSDs nutzen das Schema `/dev/nvmeXnYpZ`, z. B. `/dev/nvme0n1p1`
      - Die Nummerierung von Laufwerken und Partitionen erfolgt nach der Reihenfolge ihrer Erkennung durch das System.
